Level 1 — Dashboard (the roof)

Dashboard rework
dashboard.html
The top-level container. Everything lives under this. Shows "My Orgs" grid — each org is a clickable card. For admin: shows ALL orgs + system services. For customers: shows their org(s) only.
Cards are clickable → each opens that org's detail view
[+ New Org] button → enters onboard flow

Level 2 — Inside an Org

Org Home rework
org.html?slug=...
Single org view. Shows agents roster, recent tasks summary, workspace overview, health status. Clickable sub-navigation into Queue, Flows, Configure.
Tabs/cards link to: Queue, Flows, Configure, Agents
Queue exists
queue.html?org=...
Live work queue scoped to this org. Tasks with status flow, agent badges, auto-heal visualization. Clickable task rows expand detail.
Flows exists
flows.html?org=...
Visual workflow canvas for this org. Drag-and-drop agent assignments, trigger chains. Customers see their flows; admin sees all.
Configure exists
configure.html?org=...
Role builder for this org. Assign agent vs. human to each role. Ethics guardrails, autonomy levels. Clickable role cards toggle assignment.

Key Architecture Decisions

Decision Before After
Dashboard position Peer of other pages Roof — everything is under it
Org cards Decorative status display Clickable → drill into org detail
Queue, Flows, Configure Global/standalone pages Scoped to an org (via ?org= param)
Demo mode Separate ?demo=true behavior Pre-seeded sandbox org, same page structure
Onboard flow exit Dead end at Configure Returns to Dashboard with new org card
Admin distinction ?admin=1 URL param Auth-based; admin sees all orgs + system panels
Everything clickable Some cards are display-only Cards, task rows, agent badges — all drill down

Landing
Marketing page. "Try the demo" CTA is prominent. No login required to start exploring.
Demo Dashboard
Pre-seeded with "Acme Inc." org card. Card is clickable. Persistent CTA: "Start building your own →" leads to signup.
Demo Org Home
Agents are running, tasks are in progress. Can click into Queue (sample tasks), Flows (view-only canvas). Configure is hidden — they haven't earned that yet.
Demo Queue
Sample tasks with agents working. Auto-heal fires. Feels alive. "This could be your team" energy.

What they can't do: Create orgs, configure roles, save anything, see system services. Everything resets on refresh.

Dashboard
Shows only their org(s). Org cards are clickable. [+ New Org] to create another. No system services panel, no other customers' orgs.
Org Home
Their agents, their tasks, their workspace. Full read/write. Tabs: Queue, Flows, Configure. Each is clickable and scoped to this org.
Queue
Live tasks for their org. Can see agent assignments, status, auto-heal. Click task rows for detail. Can manually trigger tasks.
Flows
Full workflow canvas. Drag-drop agents, create trigger chains, edit automations. Saves to their org.
Configure
Role builder with agent/human toggles. Autonomy slider. Ethics lead always-on. Changes apply to their org immediately.

What they can't see: Other customers' orgs, global services, CS backdoor, Theme Studio.

Admin View (You)

Everything customers see, plus: all orgs, system services, CS tools.

Dashboard (all orgs)
Every org across all customers. Search, filter by status/owner. Click any org card to drill in. "Review my orgs" section at top for your own.
Extra panels: Global Services status, infrastructure health, total agent/human counts across system
CS Backdoor
Agent-mediated entry into customer orgs for support. No raw PII exposed — agent surfaces only relevant context. Audit trail on every access.
Global Queue
All tasks across all orgs. Filter by org, agent, status. Useful for spotting system-wide issues or stuck tasks.
Theme Studio
CSS token editor, design system management. Not a customer feature — purely operational tooling.

Admin distinguisher: Auth-based, not URL-param based. Your account is flagged as admin. No ?admin=1 needed.
