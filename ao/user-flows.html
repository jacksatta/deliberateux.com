Navigation Issues Found

Fix 3 different nav systems across 12 pages

Customer pages use a sticky topbar with varying link sets. Admin tools (tracker, sitemap, user-flows) each have custom topbars. App shell has a sidebar. Onboard has a minimal back-link. This is why you feel disoriented — there's no consistent chrome.

Fix No "you are here" indicator

The nav links don't highlight the current page. Queue, Flows, and Configure all look the same in the nav bar. You have to read the page title to know where you are.

Simplify Irrelevant links persist across audiences

"Get started" CTA shows for existing users who've already onboarded. Admin-only links show for everyone (when gated). The nav should adapt to who you are.

Add No org context in nav

When you're inside an org's queue or flows, the nav doesn't show which org you're in. You lose context of your scope.

Add Progress indicator for new-user flow only exists on onboard

The setup journey is Onboard → Bootup → Configure → Dashboard (4 pages), but only onboard shows progress pips. The user doesn't know they're 50% done after bootup.

Simplify Admin tools have no way back

Tracker, sitemap, and user-flows pages don't link back to the dashboard or each other. You have to use the browser back button or edit the URL.
