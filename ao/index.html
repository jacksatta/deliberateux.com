How it works

AO is a protocol, not a platform. Each agent boots from a shared spec, reads handoffs from other agents, runs its tasks within a time budget, and writes structured output for the next agent in the chain.

1 Boot sequence

Each agent loads its role spec, reads STATUS.md for current state, and checks for handoffs addressed to it. No shared memory — all context is explicit.

2 Time budgets

Every agent run has a wall-clock budget. If it can't finish, it writes a partial handoff and yields. No runaway loops, no infinite retries.

3 Handoff chain

Agents communicate via markdown handoff files — structured, auditable, and human-readable. The EA routes work; specialists execute.

4 Quality gates

QA reviews every outward-facing output before it ships. UXR audits interfaces. Nothing reaches users without passing at least one gate.

The seven agents

Each agent has a defined role, tools, and quality bar. They run independently but coordinate through the handoff protocol.

Executive Assistant

EA · orchestrator

Triages work, routes handoffs, synthesizes cross-agent status, and escalates blockers to the human. The hub of the network.

Marketing Lead

Mkt · marketing

Competitive scans, positioning, copy drafting, and launch planning. Blocked on pricing decisions from the human.

Dev Lead

Dev · dev lead

Implements features, fixes bugs, manages the build pipeline. Receives P0 briefs from UXR and QA for priority work.

UX Researcher

UXR · research

Heuristic evaluations, accessibility audits, and usability analysis. Writes briefs for Dev with severity ratings and evidence.

QA Guardian

QA · guardian

End-to-end testing, regression checks, and release gates. Nothing ships without QA sign-off.

Biz Dev

Biz · biz dev

Revenue model research, pricing analysis, payment platform evaluation, and distribution strategy.

Customer Service

CS · support

Inbox triage, bug reports from users, feature request routing, and reply drafting. Escalates to EA when blocked.

The AOS v2 protocol

Every agent follows the same lifecycle. The protocol is file-based, human-auditable, and designed for async coordination without shared state.

Boot → Read STATUS → Check handoffs → Execute tasks → Write output → Update STATUS

Handoff format

Markdown files with structured frontmatter: agent ID, timestamp, priority, and target agent. Human-readable and git-trackable.

STATUS.md

Single source of truth for the entire agency. Each agent reads it on boot and updates its section on completion. The EA synthesizes cross-agent state.

Self-healing

If an agent fails mid-run, its partial output is preserved. The next run detects incomplete work and resumes or re-routes.

Human-in-the-loop

The human approves pricing, strategy, and release decisions. Everything else is autonomous. The EA surfaces decision points proactively.
